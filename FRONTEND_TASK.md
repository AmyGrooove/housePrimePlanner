# FRONTEND TASK: Web-конструктор планировки дома

## 1. Цель проекта

Нужно реализовать frontend-приложение для проектирования планировки дома/квартиры.

На текущем этапе требуется только frontend без авторизации, backend, базы данных и real-time синхронизации.

Главная задача MVP: сделать удобный 2D-редактор, где пользователь может:

- создавать проект планировки;
- создавать комнаты отдельно;
- рисовать стены линиями;
- задавать точные размеры стен;
- использовать шаблоны объектов, например двери и окна;
- собирать общий план из отдельных комнат;
- перемещать комнаты и цеплять их друг к другу;
- работать в интерфейсе, похожем по ощущениям на Figma / Miro.

## 2. Технологический стек

Использовать:

- Next.js 15 App Router;
- React 19;
- TypeScript;
- pnpm;
- SCSS Modules или CSS Modules;
- Effector для состояния приложения;
- Zod для валидации данных;

- Zustand использовать не нужно, если используется Effector.

Для canvas / 2D-редактора выбрать одну из популярных библиотек:

### Предпочтительный вариант

- `konva`
- `react-konva`

Причина: удобно работать с canvas, объектами, drag-and-drop, zoom, pan, слоями, трансформациями и интерактивными фигурами.

### Дополнительные библиотеки

Можно использовать:

- `nanoid` для id;
- `clsx` или `classnames` для классов;
- `lucide-react` для иконок;
- `@floating-ui/react` для popup/tooltip/context menu;
- `react-hook-form` для форм;
- `zod` для схем;
- `effector` и `effector-react` для состояния;
- `sonner` для уведомлений.

Не использовать тяжелые UI-киты на старте. Интерфейс лучше собрать вручную.

## 3. Основные сущности

### Project

Проект планировки.

```ts
type Project = {
  id: string
  name: string
  rooms: Room[]
  objectTemplates: ObjectTemplate[]
  createdAt: string
  updatedAt: string
}
```

### Room

Комната, которая создается отдельно и потом добавляется в общий план.

```ts
type Room = {
  id: string
  name: string
  position: Point
  walls: Wall[]
  placedObjects: PlacedObject[]
}
```

### Wall

Стена или сегмент стены.

```ts
type Wall = {
  id: string
  start: Point
  end: Point
  thickness: number
}
```

### Point

```ts
type Point = {
  x: number
  y: number
}
```

### ObjectTemplate

Шаблон объекта. Например: дверь, окно, входная дверь.

```ts
type ObjectTemplate = {
  id: string
  type: ObjectTemplateType
  name: string
  width: number
  height: number
  thickness: number
  metadata?: Record<string, unknown>
}

type ObjectTemplateType = 'door' | 'window' | 'furniture' | 'custom'
```

### PlacedObject

Объект, размещенный на плане.

```ts
type PlacedObject = {
  id: string
  templateId: string
  position: Point
  rotation: number
  attachedWallId?: string
}
```

## 4. Основной функционал MVP

### 4.1. Создание шаблонов объектов

Нужно сделать интерфейс для создания и редактирования шаблонов.

На старте поддержать типы:

- дверь;
- окно;
- кастомный объект.

Для каждого шаблона пользователь задает:

- название;
- тип;
- ширину;
- высоту;
- толщину.

Пример шаблонов:

- Дверь 900x2100;
- Мини-дверь 720x2100;
- Входная дверь 980x2070;
- Окно 1870x1610.

Размеры хранить в миллиметрах.

---

### 4.2. Редактор комнаты

Пользователь должен иметь возможность создать комнату отдельно.

В редакторе комнаты нужно реализовать:

- canvas-сцену;
- инструмент рисования стен;
- отображение стен;
- перемещение точек стен;
- ввод точной длины стены;
- отображение длины стены рядом с линией;
- толщину стены;
- привязку углов к шагу 45 градусов;
- zoom;
- pan;
- сетку;
- snap к сетке.

Комната может быть не только прямоугольной. Нужно поддержать произвольный контур из линий.

---

### 4.3. Рисование стен

Стены рисуются линиями.

Поведение:

1. Пользователь выбирает инструмент `Wall`.
2. Кликает по canvas, чтобы поставить начальную точку.
3. Ведет мышью до конечной точки.
4. Линия должна поворачиваться с шагом 45 градусов.
5. Повторный клик фиксирует конец линии.
6. Следующая линия может начинаться из предыдущей точки.
7. Пользователь может завершить рисование через `Escape` или отдельную кнопку.

Нужно поддержать ручной ввод длины выбранной стены.

При вводе новой длины:

- start-точка остается на месте;
- end-точка пересчитывается по текущему углу стены;
- длина меняется точно.

---

### 4.4. Общий конструктор плана

После создания комнат пользователь должен видеть общий экран плана.

На нем можно:

- видеть список комнат;
- добавлять комнаты на общий canvas;
- перемещать комнаты;
- вращать комнаты с шагом 45 градусов;
- цеплять комнаты друг к другу по стенам или точкам;
- включать/выключать snap;
- включать/выключать сетку.

На старте достаточно простого snap:

- snap по углам комнат;
- snap по ближайшим точкам стен;
- визуальная подсказка при приближении.

Не нужно сразу делать сложное объединение стен. Достаточно, чтобы комнаты визуально стыковались и сохраняли позиции.

---

### 4.5. Объекты на плане

Нужно дать возможность размещать шаблоны объектов на стенах.

На MVP достаточно:

- выбрать шаблон объекта;
- кликнуть по стене;
- объект появляется на стене;
- объект можно двигать вдоль стены;
- объект наследует угол стены;
- объект можно удалить.

Двери и окна отображать условно:

- дверь как проем + дуга открывания;
- окно как тонкий прямоугольник внутри стены.

## 8. Важные требования к коду

- Писать на TypeScript.
- Использовать стрелочные функции.
- Экспортировать функции через `export { functionName }`.
- Не использовать однобуквенные переменные.
- Не использовать непонятные сокращения в названиях.
- Названия переменных должны быть полными и читаемыми.
- Компоненты должны быть небольшими.
- Canvas-логику выносить в отдельные функции.
- Геометрические расчеты выносить в `shared/lib/geometry`.
- Не смешивать UI и математические расчеты.
- Не использовать `any`, если можно описать тип.
- Не запускать тесты, линтеры, форматтеры и typecheck по умолчанию.

## 9. Геометрические функции

Создать модуль:

```txt
/src/shared/lib/geometry
```

Нужны функции:

- расчет длины линии;
- расчет угла линии;
- snap точки к сетке;
- snap угла к 45 градусам;
- пересчет end-точки по start-точке, углу и длине;
- поиск ближайшей точки;
- поиск ближайшей стены;
- расчет позиции объекта на стене.

Пример функций:

```ts
const getDistanceBetweenPoints = (firstPoint: Point, secondPoint: Point): number => {}

const getAngleBetweenPoints = (startPoint: Point, endPoint: Point): number => {}

const snapAngleToStep = (angle: number, step: number): number => {}

const getPointByAngleAndDistance = (startPoint: Point, angle: number, distance: number): Point => {}
```

```txt
1 px = 10 mm
```

```ts
const convertMillimetersToPixels = (millimeters: number): number => {}
const convertPixelsToMillimeters = (pixels: number): number => {}
```

## 11. Persistence на MVP

Backend пока не нужен.

Данные сохранять в `localStorage`.

Нужно сохранять:

- проект;
- комнаты;
- стены;
- шаблоны объектов;
- размещенные объекты;
- настройки редактора.

Использовать Zod для проверки данных при загрузке из `localStorage`.

Интерфейс должен быть чистым, спокойным и похожим на профессиональный редактор.

Ориентиры:

- Figma;
- Miro;
- FigJam;
- простые CAD-редакторы.

Цвета:

- светлый фон рабочей области;
- мягкая сетка;
